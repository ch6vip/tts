API概述

TTS服务API提供了简单而强大的方式将文本转换为自然语音。我们支持多种语言和声音，并允许您调节语速、语调以适应不同场景需求。

基础URL: {{.BasePath}}

所有API请求均使用HTTP协议，返回标准HTTP状态码表示请求结果。

文本转语音 API

端点

HTTP 
GET /tts

同时支持POST请求，端点为 POST /tts，参数以JSON格式在请求体中提供。

参数

参数 类型 必选 描述
t string 是 要转换的文本（需要进行URL编码）
v string 否 语音名称，使用short_name格式，默认: {{.DefaultVoice}}。可通过/voices接口获取所有可用语音
r string 否 语速调整，范围: -100%到100%，默认: {{.DefaultRate}}。正值加快语速，负值减慢语速
p string 否 语调调整，范围: -100%到100%，默认: {{.DefaultPitch}}。正值提高语调，负值降低语调
o string 否 输出音频格式，默认: {{.DefaultFormat}}。详见下方支持的格式列表
s string 否 情感风格，可用值取决于所选语音的style_list属性。例如："cheerful"、"sad"等

示例请求

cURL 
curl "{{.BasePath}}/tts?t=%E4%BD%A0%E5%A5%BD%EF%BC%8C%E4%B8%96%E7%95%8C&v=zh-CN-XiaoxiaoNeural&r=0%25&p=0%25"
JavaScript 
const text = encodeURIComponent("你好，世界");
const voice = "zh-CN-XiaoxiaoNeural";
const rate = "0%";
const pitch = "0%";

fetch(`{{.BasePath}}/tts?t=${text}&v=${voice}&r=${rate}&p=${pitch}`)
  .then(response => response.blob())
  .then(blob => {
    const audioUrl = URL.createObjectURL(blob);
    const audio = new Audio(audioUrl);
    audio.play();
  });
Python 
import requests
import urllib.parse

text = urllib.parse.quote("你好，世界")
voice = "zh-CN-XiaoxiaoNeural"
rate = "0%"
pitch = "0%"

url = f"{{.BasePath}}/tts?t={text}&v={voice}&r={rate}&p={pitch}"
response = requests.get(url)

if response.status_code == 200:
    with open("output.mp3", "wb") as f:
        f.write(response.content)
    print("音频文件已保存为 output.mp3")
else:
    print(f"请求失败: {response.status_code}")
Go 
package main

import (
    "fmt"
    "io"
    "net/http"
    "net/url"
    "os"
)

func main() {
    text := url.QueryEscape("你好，世界")
    voice := "zh-CN-XiaoxiaoNeural"
    rate := "0%"
    pitch := "0%"

    apiURL := fmt.Sprintf("{{.BasePath}}/tts?t=%s&v=%s&r=%s&p=%s", text, voice, rate, pitch)
    
    resp, err := http.Get(apiURL)
    if err != nil {
        fmt.Printf("请求失败: %v\n", err)
        return
    }
    defer resp.Body.Close()

    if resp.StatusCode == http.StatusOK {
        file, err := os.Create("output.mp3")
        if err != nil {
            fmt.Printf("创建文件失败: %v\n", err)
            return
        }
        defer file.Close()

        _, err = io.Copy(file, resp.Body)
        if err != nil {
            fmt.Printf("保存文件失败: %v\n", err)
            return
        }

        fmt.Println("音频文件已保存为 output.mp3")
    } else {
        fmt.Printf("请求失败，状态码: %d\n", resp.StatusCode)
    }
}

POST 请求示例 (JSON)

cURL 
curl -X POST "{{.BasePath}}/tts" \
-H "Content-Type: application/json" \
-d '{
    "t": "这是通过POST请求发送的一段长文本。这种方式更适合处理大量文本内容。",

    "r": "5%",

响应

返回音频文件，内容类型取决于请求的输出格式。正常响应状态码为200。

错误响应

如果请求参数有误或服务出现问题，将返回对应的HTTP错误码和错误消息。

状态码 描述
400 参数错误或缺失必要参数
404 请求的资源不存在
500 服务器内部错误

获取可用语音 API

端点

HTTP 
GET /voices

参数

参数 类型 必选 描述
locale string 否 筛选特定语言的语音，例如：zh-CN（中文）、en-US（英文）
gender string 否 筛选特定性别的语音，可选值：Male（男性）、Female（女性）

示例请求

cURL 
curl "{{.BasePath}}/voices?locale=zh-CN&gender=Female"

响应

返回JSON格式的可用语音列表：

JSON 
[
  {
    "name": "Microsoft Server Speech Text to Speech Voice (zh-CN, XiaoxiaoNeural)",
    "display_name": "Xiaoxiao",
    "local_name": "晓晓",
    "short_name": "zh-CN-XiaoxiaoNeural",
    "gender": "Female",
    "locale": "zh-CN",
    "locale_name": "中文(中国)",
    "style_list": ["cheerful", "sad", "angry", "fearful", "disgruntled"]
  },
  ...
]

响应字段说明：

  • name：语音的完整名称
  • display_name：显示用名称（拉丁字符）
  • local_name：本地化名称
  • short_name：简短名称（用于API调用的v参数）
  • gender：性别（Male或Female）
  • locale：语言代码
  • locale_name：语言本地化名称
  • style_list：支持的情感风格列表（如有）

兼容OpenAI接口 API

语音合成 (Speech)

HTTP 
POST /v1/audio/speech

请求头中需要包含 Authorization 字段，尽管当前服务实现中此字段为可选，但建议遵循OpenAI规范。例如: Authorization: Bearer sk-your-token。

请求体 (JSON)

参数 类型 必选 描述
model string 是 模型名称，当前支持 "tts-1" 和 "tts-1-hd"。
input string 是 要转换的文本内容
voice string 是 声音名称，使用Microsoft语音格式，例如：ja-JP-KeitaNeural、zh-CN-XiaoxiaoNeural
speed number 否 语速调整，范围: 0.5到2.0，默认: 1.0
response_format string 否 输出音频格式，默认: "mp3"。支持的格式有 "mp3", "opus", "aac", "flac", "wav", "pcm"。

示例请求

cURL 
curl -X POST "{{.BasePath}}/v1/audio/speech" \
  -H "Authorization: Bearer sk-your-token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tts-1",
    "input": "你好，世界！",
    "voice": "zh-CN-XiaoxiaoNeural"
  }'

另一个示例（带速度调整）

cURL 
curl -X POST "{{.BasePath}}/v1/audio/speech" \
  -H "Authorization: Bearer sk-your-token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tts-1-hd",
    "input": "こんにちは、世界！",
    "voice": "ja-JP-NanamiNeural",
    "speed": 1.2,
    "response_format": "opus"
  }'

响应

返回音频文件，内容类型取决于请求的输出格式。正常响应状态码为200。

错误响应

如果请求有误，将返回JSON格式的错误信息：

JSON 
{
  "error": {
    "message": "错误信息描述",
    "type": "错误类型",
    "code": "错误代码"
  }
}

支持的输出格式

格式名称 描述
audio-16khz-32kbitrate-mono-mp3 MP3格式，16kHz, 32kbps
audio-16khz-64kbitrate-mono-mp3 MP3格式，16kHz, 64kbps
audio-16khz-128kbitrate-mono-mp3 MP3格式，16kHz, 128kbps
audio-24khz-48kbitrate-mono-mp3 MP3格式，24kHz, 48kbps
audio-24khz-96kbitrate-mono-mp3 MP3格式，24kHz, 96kbps
audio-24khz-160kbitrate-mono-mp3 MP3格式，24kHz, 160kbps
riff-16khz-16bit-mono-pcm WAV格式，16kHz
riff-24khz-16bit-mono-pcm WAV格式，24kHz

系统监控 API

获取性能指标

HTTP 
GET /metrics

返回一个JSON对象，包含详细的性能指标，如TTS请求数、成功率、延迟、缓存命中率和系统内存使用情况。

示例响应:

JSON 
{
    "tts": {
        "requests": 100,
        "success": 98,
        "errors": 2,
        "success_rate": 98.0,
        "latency": { "avg": "500ms", "max": "1.2s", "min": "300ms" }
    },
    "cache": { "hits": 80, "misses": 20, "hit_rate": 80.0, "total_size": 10240 },
    "worker_pool": { "total_jobs": 50, "errors": 1 },
    "system": {
        "memory": { "alloc_mb": 50, "total_alloc_mb": 200, "sys_mb": 500, "num_gc": 10 },
        "goroutines": 15
    },
    "timestamp": "2025-10-18T20:24:00Z"
}

响应字段说明:

  • tts (语音合成)
    • requests: 请求总数
    • success: 成功次数
    • errors: 失败次数
    • success_rate: 成功率 (%)
    • latency: 延迟 (avg: 平均, max: 最大, min: 最小)
  • cache (缓存)
    • hits: 命中次数
    • misses: 未命中次数
    • hit_rate: 命中率 (%)
    • total_size: 缓存总大小 (字节)
  • worker_pool (工作池)
    • total_jobs: 处理的任务总数
    • errors: 任务错误数
  • system (系统)
    • memory: 内存使用情况 (MB)
    • goroutines: 当前协程数
  • timestamp (时间戳): 指标生成的时间

重置性能指标

HTTP 
POST /metrics/reset

重置所有累计的性能指标。成功后返回一条确认消息。

示例响应:

JSON 
{
    "message": "Metrics reset successfully"
}

健康检查

HTTP 
GET /health

检查服务的健康状况。如果服务正常，返回HTTP 200状态码；如果服务存在问题（例如错误率过高），则返回HTTP 503。

示例响应 (健康):

JSON 
{
    "healthy": true,
    "reason": "ok",
    "timestamp": "2025-10-18T20:24:00Z"
}
